Shared Composables (client/src/composables/)

useFilters.js
Module-level singleton refs for period, location, category, and status. All views and FilterBar share the same reactive state. getCurrentFilters() serialises to API params.
useI18n.js
EN/JA locale lookup via nested key paths. Exposes t(key), currentCurrency (USD/JPY), and helper methods for translating warehouse and product names.
useAuth.js
User profile state, task list management, and language/currency switching. Wired to ProfileMenu and LanguageSwitcher components in the header.

Filter Propagation Pattern

  • FilterBar.vue binds to singleton refs in useFilters via v-model
  • Every view calls useFilters() and watches the same refs → watch([selectedLocation, selectedCategory, ...], loadData)
  • On change, getCurrentFilters() serialises the active refs to an object passed to api.*
  • Each api.* method appends only non-'all' values as URL query params via URLSearchParams
  • Backend apply_filters() and filter_by_month() do case-insensitive matching, skip when value is absent or 'all'

Known Gaps (production readiness)

  • No auth — add JWT/OAuth before any external access
  • No database — JSON files don't persist mutations
  • CORS wildcard — restrict to known origins in prod
  • No pagination — all data loaded per request
  • No rate limiting — add nginx or FastAPI middleware
  • No HTTPS — TLS termination required at load balancer

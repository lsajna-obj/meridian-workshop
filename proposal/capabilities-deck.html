R2 — Restocking recommendations

What your operations
team has been asking for

How it works

Operator enters a budget ceiling. The system ranks purchase order recommendations by urgency — stock vs. demand — and fits them within the budget.


What you see

Each recommendation shows current stock, forecasted demand, suggested order quantity, and estimated cost. Filterable by warehouse.

How it's built

New /api/restocking backend endpoint. New Vue 3 view following existing codebase patterns. No database changes — works with the current data layer.
